VR WEAPONS

Description:

I already had a fairly good weapon system from one of my VR projects, but since I didn't like how mechanical it felt i decided to write a new one using mostly physics based operations.

This new system uses Unity's physics engine to handle all collision and moving parts of the weapons, and firing is done using projectiles instead of hitscan.

The 3D models for this had to be custom-made and are mostly accurate to the size of the real thing


Technical stuff:

The game was written and built in unity.

The basis for the VR setup was done using the SteamVR addon.

Since bullets go faster than the frame-rate they still use raycast's to retrace their path to check for collision.

Moving parts are usually frozen in place when not in use.

Script is used to check the state of the gun and reading VR inputs for firing.

CURRENTLY IN DEMO:
-Basic FPS controls
-Physics & AI targets

PROBLEMS IN DEMO:
-Bolt action rifle is not included due to physics problems
-pistol sometimes doesn't load magazine properly

FIGHTING GOLD

Description:

This is my biggest project that I'm currently working on.

It's a 3D fighting game using a combat system similar to For honor by Ubisoft and mostly inspired by Guilty Gear made by ArcSystemWorks.

Compared to the games it's inspired by it has a lot of differences: the attacks and combat is way faster than For honor and it cant be compared to Guilty Gear because it's a 2D game instead of 3D like mine.

Adding to these differences I'm busy with making a game mode which I've never seen done before in the way I'm planning.


Technical stuff:

I'm using Unity3D for this project.

I'm making all the 3D models, animations, UI, Effects and script for AI, combat and game management myself.

The most technical part so far was either the shader for the center line and special zones. Or the blood effect which I want to have stick to the map and stay there until the match is done.

Other than that it's mostly just attack lists, stats for characters, and AI written using Unity's AI system as a base

MOBILE RACING GAME

Description:

For a school project we needed to make a game using touch screen controls.

I decided to make a racing game mostly copying Mario kart DS because the controls in that game are very simple but very good.


Technical stuff:

The game was written and built in unity.

All 3D models & maps were custom-made by myself (except map 1) using blender.

The ingame camera is put through a filter to make the game look pixelated.

Movement for karts is done using a rigidbody.

AI gets next checkpoint from list and detects direction via raycasts.


CURRENTLY IN DEMO:
-Touch & Keyboard controls
-3 maps to race on
-4 karts to choose from

PROBLEMS IN DEMO:
-No pause menu in game
-AI opponents not active in game

STAR-WIZARDS

Description:

Made as a custom project from school with the help of another student.

I did most of the programming and 3D modeling for this game.

The game is an infinite level with score save files and constant action

This game is still in development, we are working on a mission mode


Technical stuff:

The game was written and built in unity.

All 3D models are custom made in blender.

The level is randomly generated and enemy spawning/difficulty is dependent on the score.

The collision system uses raycasts instead of colliders due to some physics problem.


CURRENTLY IN GAME:
-8 enemy types
-both open flying and guided section
-Score saves

PROBLEMS IN GAME:
-No mission mode
