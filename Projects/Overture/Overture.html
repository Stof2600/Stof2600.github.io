LETS MAKE A GAME

My younger brother asked me to help in making a game he had designed

After looking at the design document it was very similar to Guilty Gear 2: Overture

However the strategy system had been modernized and he had created his own characters and plot

It seemed fun to work on and I had no other project at the time so I accepted


GAMEPLAY

The player has two responsablilities while playing

Firstly is fighting: killing enemy players and destroying their base to win

This follows a basic 3D fighting game style using direction inputs and combos


Secondly the strategy menu: buying and commanding your units to face of against the enemies units

In this real-time strategy part you need to both capture and defend the control points on the map


winning in the strategy can give you items and skills to assint in the fighting

At the same time fighting properly allows your units to advance and give the player more chances to strike the enemy base


STRATEGY SYSTEM

It uses unity NavMesh to move all the units, to optimize it a bit only one teams AI is updated on a frame

The small spheres are capture units and always move to the next control point, their AI path only need to be set on spawn

The units that can be bought by the player can be assigned to groups

Every group has a movement target, this can either be a control point or a base

If the group encounters an enemy group they will start combat, this currently has no animation

Once the group reaches their target they will assist in its capture or fight off any enemy capture units

If captured a control point will award the player with "COMMAND"

COMMAND can be used to buy troops, items, skills and other things to assist the player

In order for the player to not instantly destroy the other teams base the units need to reach it first to break the shield


UNITS

Currently there are 4 purchasable units

Knights: medium armor units with basic stats

Archers: Slower and weaker then knights they are easy to take out but have ranged attacks

Cavalry: fast units but with light armor

Heavy Knights: slow but with good defence

Each unit also has a counter or a unit type its weak to

If you spot the opponent using a single type of unit you can invest in the counter to it


FIGHTING

The fighting system is currently not finished, but its very similar to most 2D fighting games with frame timers and hitboxes


OTHER STUFF

I wanted to mention the scrolling text in the background of the title screen because it think its funny

We designed it based on Cowboy Bebop's similar "just put text everywhere" approach

The text spawning and moving isn't interesting but the content itself is

It uses a random line from the games code to display, ignoring empty lines and the "{ }" brackets in the code

Technically if you read it for long enough you could get all the lines of code to remake the game
